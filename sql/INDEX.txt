SECURITY POLICIES AT A GLANCE
================================================================================

ROLE ACCESS PATTERNS:

admin_polimata (System Administrator)
   • MRC: SELECT ALL, INSERT ALL, UPDATE ALL, DELETE ALL
   • Clients: SELECT ALL, DELETE ALL
   • Projects: SELECT ALL, DELETE ALL
   • All tables: Full CRUD
   → Use: Polímata internal team only

gestor_cliente (Client Manager)
   • MRC: SELECT/INSERT/UPDATE own client only
   • Projects: SELECT/INSERT/UPDATE own client only
   • Clients: SELECT own only
   • Areas, Reviews: CRUD own client only
   • Delete: No access
   → Use: Client-side managers

consultor_polimata (Consultant)
   • MRC: SELECT/INSERT/UPDATE assigned projects only
   • Clients: No direct access (via projects)
   • Projects: SELECT assigned only
   • Areas, Reviews: CRUD assigned projects only
   • Delete: No access
   → Use: Consultants working on specific projects

usuario_cliente (Client User)
   • MRC: SELECT own client only
   • Everything else: No access
   • Operations: Read-only
   → Use: Stakeholders, read-only viewers

ISOLATION KEYS (Data Segmentation)
================================================================================

Level 1 - TENANT ISOLATION (cliente_id)
   Tables: perfis, clientes, sistemas, responsaveis
   Scope: Users belong to one client
   Breach impact: CRITICAL (cross-client data leak)
   Protection: RLS on cliente_id column

Level 2 - PROJECT ISOLATION (projeto_id)
   Tables: projetos, areas, mrc, revisoes, notificacoes
   Scope: Projects belong to one client
   Breach impact: HIGH (cross-project data leak)
   Protection: RLS on projeto_id + cliente_id

Level 3 - CONSULTANT ASSIGNMENT (perfis_projetos)
   Tables: perfis_projetos
   Scope: Consultants assigned to specific projects
   Breach impact: MEDIUM (unauthorized project access)
   Protection: RLS on perfil_id + projeto_id

TROUBLESHOOTING MAP
================================================================================

Problem: "Permission denied" error
   → Solution: Check DEPLOYMENT_GUIDE.md "Common Issues" section 1
   → Check: User exists in perfis table with correct papel

Problem: Queries return no rows
   → Solution: Check DEPLOYMENT_GUIDE.md "Common Issues" section 2
   → Check: cliente_id or projeto_id matches user's assignment

Problem: Slow queries
   → Solution: Check DEPLOYMENT_GUIDE.md "Performance Optimization"
   → Run: EXPLAIN ANALYZE SELECT ...
   → Add: Recommended indexes

Problem: Junction table errors
   → Solution: Check DEPLOYMENT_GUIDE.md "Common Issues" section 4
   → Verify: Tables perfis_projetos and permissoes_area exist

Problem: Rollback needed
   → Solution: Check DEPLOYMENT_GUIDE.md "Rollback Plan"
   → Run: ALTER TABLE ... DISABLE ROW LEVEL SECURITY
